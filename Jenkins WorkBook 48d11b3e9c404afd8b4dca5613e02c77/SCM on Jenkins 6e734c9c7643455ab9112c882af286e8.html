SCM on Jenkins

  • Go to + New Item
  • And create a Freestyle project

  • The on the configuration of project go to Source Code Management and select Git
  • In this case we are a going to work with a bash shell script in a remote github repo
#!/bin/bash
echo "this script is from github"
NUMBER=$(($NUMBERONE + $NUMBERTWO))
echo "$NUMBER"
  • Paste url of the repo as if you are cloning the repo

Note: Make sure you have installed git on the machine where Jenkins is running

Go to Build steps and make the script executable and run it

  • Go to General and check “This project is parameterized”
  • Add a the string parameters with the name used by the script (NUMBERONE, NUMBERTWO) and assign them a value
  • Save the project and the run the build

On the workspace you will see all of the github files

Navigate to the console output and you will see the result of the script

If you want to execute the project every time you make a commit to the repo

  • Go to Dashboard → Manage Jenkins → Configure System → GitHub → Add GitHub Server
  • Copy your access token from GitHub, go to setting → Developer Settings → Personal Access Token
  • Then on Jenkins Server click on add credentials → Jenkins
  • Kind: Secret text and on secret copy your personal access token from github, finally give it a name on the id section and a optional description and click on add

  • Go back to your project → configuration → Build triggers
  • Select GitHub hook trigger for GITScm polling
  • And Save
  • Modify the script by adding a line at the end like: echo "This is the end of the script”
  • commit it and push it to the remote repo
  • You will see a build on the Jenkins server when the code is pushed
